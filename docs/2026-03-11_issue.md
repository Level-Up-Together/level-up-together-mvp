# 2026-03-11 E2E 테스트 이슈 분석

## 이슈 1: `알 수 없는 dataField: guildJoinCount` — 업적 달성 불가 버그

**심각도**: HIGH (기능 미작동)

**증상**: `WARN i.p.l.g.a.s.UserStatsCheckStrategy : 알 수 없는 dataField: guildJoinCount`

**원인**: DB에 업적 7개(id 50~56, "길드 N회 가입")가 `check_logic_data_source=USER_STATS`, `check_logic_data_field=guildJoinCount`로 설정되어 있으나:
- `UserStats` 엔티티에 `guildJoinCount` 필드 없음
- `UserStatsCheckStrategy` switch에 `guildJoinCount` 케이스 없음
- `UserStatsCounterEventListener`에서 `GuildJoinedEvent` 수신 시 카운트 증가 로직 없음

**영향**: 길드 가입 관련 업적(50~56)이 절대 달성되지 않음

**수정 범위**:
1. `UserStats` 엔티티에 `guildJoinCount` 필드 추가 + DDL
2. `UserStatsCheckStrategy`에 `guildJoinCount` 케이스 추가
3. `UserStatsCounterEventListener`에서 `GuildJoinedEvent` 수신 → `guildJoinCount` 증가
4. 테스트 코드 추가

## 이슈 2: `ClassCastException: MissionCategoryResponse` — DevTools 클래스로더 충돌

**심각도**: LOW (로컬 개발 환경 한정, 자동 복구됨)

**증상**:
```
ClassCastException: class MissionCategoryResponse cannot be cast to class MissionCategoryResponse
(loader 'app' vs loader RestartClassLoader)
```

**원인**: Spring Boot DevTools의 `RestartClassLoader`와 기본 `app` 클래스로더 간 충돌. Redis에 캐시된 객체가 이전 클래스로더로 직렬화된 상태에서 DevTools 재시작 후 다른 클래스로더로 역직렬화 시도.

**영향**: 로컬 개발 환경에서만 발생. prod에는 DevTools 미포함. `RedisConfig`의 `CacheErrorHandler`가 손상된 캐시를 자동 evict하여 자동 복구됨.

**수정 방안** (선택):
- `META-INF/spring-devtools.properties`에 프로젝트 패키지를 restart classloader에 포함하도록 설정
- 또는 현행 유지 (CacheErrorHandler가 자동 복구하므로 실질적 영향 없음)

## 이슈 3: `손상된 캐시 엔트리 삭제 완료` — 이슈 2의 후속 처리

**심각도**: INFO (정상 동작)

**증상**: `INFO i.p.global.config.RedisConfig : 손상된 캐시 엔트리 삭제 완료 - cache: userTitleInfo, key: d842663e-...`

**원인**: 이슈 2(ClassCastException)로 인해 `CacheErrorHandler`가 작동하여 손상된 캐시를 자동 삭제한 정상 로그.

**수정**: 불필요 (정상적인 에러 핸들링 동작)
